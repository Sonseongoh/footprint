### 소개

> 직접 가서 GPS로 인증한 도시만 지도에 채워지는 여행 수집 앱. (한국 · 일본 · 태국)

### 문제 정의

> 여행 기록 앱은 이미 많습니다. Been은 가본 **나라**를 체크하고, Polarsteps는 **경로**를 타임라인으로 남깁니다. 제가 주목한 건 이 앱들이 공통으로 놓치는 질문이었습니다 — **"나는 이 나라를 얼마나 깊게 다녔나?"** 일본을 다섯 번 간 사람에게 "일본 ✓" 하나는 아무 정보가 없습니다. 그래서 수집의 단위를 나라가 아니라 **나라 안의 도시**로 잡았습니다.
>
> 두 번째 차별점은 **신뢰**입니다. 지도를 손으로 칠할 수 있는 순간 지도는 기록이 아니라 위시리스트가 됩니다. footprint는 **수동 추가를 아예 만들지 않았습니다** — 현장 GPS 인증만 지도를 채웁니다. **"이 지도는 거짓말을 못 한다"** 가 이 제품의 뼈대입니다.

### 【이미지】 주요 화면 — 5장

1. 지구본 진입 화면 (금색으로 빛나는 나라)
2. **채움 지도 (한국)** — 시 단위 금색 채움 + 상단 "채운 도시 N/M" ← *제품의 심장. 가장 크게*
3. 체크인 성공 화면 ("✓ 한국 인증됨" + 도시명)
4. 도시 화면 — 여행 공유 목록 (작성자·좋아요·사진)
5. 내 발자국 — 수집 통계 + 나라별 진행바

---

### 역할

**◆ 신뢰 모델을 UI가 아니라 DB로 강제**
- 수동 추가 없음. 현장 GPS + 행정구역 polygon 판정만 지도를 채움
- 여행 공유(공개 추천)는 **그 도시에 7일 내 체크인한 사람만** 작성 가능 — 이 규칙을 클라이언트가 아니라 **Postgres RLS + 서버 함수**가 강제. 앱을 우회해 API를 직접 호출해도 막힘
- 게스트는 **구경꾼**: 열람만 가능하고 체크인·좋아요·신고·작성은 전부 로그인 필요 (JWT의 `is_anonymous` 클레임을 검사하는 restrictive 정책)
- 【이미지】 체크인 실패 2종("지원 지역이 아니에요" / "GPS 정확도가 낮아요") + 7일 잠금 카드

**◆ 로컬 우선(offline-first) 아키텍처 — 여행은 오프라인이 기본**
- 체크인 → **SQLite 큐에 먼저 저장** + 로컬 투영 즉시 갱신(지도가 바로 채워짐) → 온라인 복귀 시 서버 동기화
- **멱등성**: 큐의 로컬 id를 그대로 서버 이벤트 id로 사용 → 재전송이 upsert가 되어 재방문 횟수 이중 집계 없음
- 재방문 누적은 클라이언트가 아닌 **DB 트리거**가 집계 (동시성 안전)
- 기록 타임라인도 마지막 동기화본을 캐시 → 비행기 모드에서 "기록 없음"으로 보이지 않음
- 【이미지】 오프라인 배너가 뜬 기록 탭 + 동기화 흐름 다이어그램

**◆ 지도 렌더링 (react-native-svg + d3-geo)**
- 나라별 행정구역 choropleth. 방문 = 금색, 미방문 = 슬레이트
- 핀치줌/팬 + **줌 레벨별 라벨 LOD**(시·도 ↔ 도시 크로스페이드) + 라벨 충돌 컬링
- 【이미지】 줌 아웃 / 줌 인 비교 2장

**◆ 사진의 이중 프라이버시**
- 체크인 사진 = **비공개 버킷 + 서명 URL** / 여행 공유 사진 = 공개 버킷
- 같은 "사진"이라도 **속한 개념의 공개성을 따라가게** 설계

**◆ 출시 요건 (UGC 정책 대응)**
- 신고(3건 누적 시 자동 숨김 트리거) + **작성자 차단**(Play UGC 정책은 신고와 차단을 **둘 다** 요구) + 계정 삭제(전 데이터 cascade) + 개인정보처리방침 웹 배포
- 【이미지】 ⋯ 신고/차단 메뉴 + 차단 관리 화면

## 📄 페이지 2

### 결과 및 성과

- **1인 end-to-end** — 문제 정의부터 설계·구현·백엔드(Postgres/RLS)·웹 배포·스토어 출시 준비까지 전 과정 수행
- 제품의 핵심 규칙(현장 인증, 7일 작성 자격, 게스트 권한)을 **전부 DB 레이어에서 강제**해, 클라이언트를 우회해도 신뢰 모델이 깨지지 않는 구조로 설계 (마이그레이션 16개)
- 여행이라는 **오프라인이 기본인 사용 맥락**을 전제로 로컬 우선 + 멱등 동기화 구조를 구현
- **기능 완성 후 8개 관점(화면별 UX 5 + 스토어 요건 3)으로 셀프 감사를 실시해 48건을 검증**하고, 출시 블로커 4건을 발견·수정
- 스토어 출시 요건(신고·차단·계정삭제·개인정보처리방침)을 직접 조사해 구현 — **"기능이 다 됐다"와 "출시할 수 있다"가 다르다**는 것을 학습

### 가설이 바뀐 지점 ⭐

- **반경 판정 → 행정구역 폴리곤**: 처음엔 "도시 중심좌표 + 반경"으로 위치 인증을 하려 했으나, 지도 채움에 어차피 경계 폴리곤이 필요했습니다. 같은 데이터를 두고 판정만 원으로 하면 대도시/소도시 반경 튜닝 문제만 생겨, **판정과 렌더링을 폴리곤 하나로 통일**했습니다.
- **채움 단위 도(道) → 시(市)**: 한국을 다른 나라처럼 admin-1(17칸)로 채웠더니 **국내 여행 몇 번이면 끝나는 빙고판**이 됐습니다. 금방 끝나는 수집은 수집욕을 못 만든다고 판단해 한국만 시 단위로 재작업했습니다(일본 47·태국 77은 유지). *"수집의 재미는 칸의 개수가 아니라 채움의 밀도에서 온다."*
- **익명 우선 → 로그인 필수**: 여행 중 마찰 최소화를 위해 "익명으로 바로 체크인, 계정은 나중에"로 구현했으나, 소셜 기능(좋아요·신고)을 붙인 뒤 설계 리뷰 중 구멍을 발견했습니다 — **익명 계정은 로그아웃만 반복하면 무한 생성**되므로, 익명 신고 3건으로 아무 글이나 숨길 수 있었습니다. **"발자국은 기기가 아니라 계정에 속한다"** 로 원칙을 다시 세우고 서버(RLS)에서 강제했으며, 뒤집은 이유를 설계 문서에 취소선으로 남겼습니다.

### 트러블 슈팅

**1. Hermes에서 d3 `geoPath`가 에러 없이 죽음**
지도가 그려지지 않는데 예외도 없었습니다. **투영만 d3에서 가져오고 경로 문자열 생성은 포기**한 뒤, 좌표를 직접 투영해 `<Polygon points>`로 렌더링해 우회했습니다. *(웹의 표준 도구가 네이티브 JS 엔진에서 그대로 돌지 않는다는 것을 학습)*

**2. 구글 로그인이 문서대로 했는데 5번 깨짐**
① 인앱 웹뷰를 구글이 정책으로 차단(`403 disallowed_useragent`) → ② 로그인 후 `localhost:3000`으로 리다이렉트(딥링크 미등록) → ③ "Unmatched Route" → ④ 무한 스피너(라우터가 URL을 먼저 소비 + 토큰이 fragment로 도착) → ⑤ "identity is already linked".
**PKCE 플로우로 전환**(코드가 fragment가 아닌 쿼리로 도착) + **앱 최상단에 딥링크 리스너**를 둬 라우팅과 무관하게 세션을 교환하도록 해결했습니다. *인증은 라이브러리 호출 한 줄이 아니라 브라우저 정책·딥링크·라우터 타이밍이 얽힌 시스템이었습니다.*

**3. 오프라인 큐가 남의 계정에 체크인을 넣을 수 있었음**
A로 오프라인 체크인 → 로그아웃 → B로 로그인 → 온라인 복귀 시 **A의 체크인이 B의 발자국**이 됐습니다(동기화가 큐의 주인이 아닌 현재 세션으로 업로드). flush 시 **큐 주인 ≠ 현재 유저면 건너뛰고**, 주인이 재로그인하면 그때 동기화하도록 수정했습니다.

**4. 비행기 모드에서 로그인 사용자가 '게스트'로 판정**
로그인 여부를 서버 호출(`getUser`)로 확인하고 있어서, 오프라인이면 대답이 없어 게스트로 오판하고 **여행 중 체크인이 차단**됐습니다. **오프라인 큐까지 만든 앱이 정작 인증 판정에서 오프라인을 잊은 것**입니다. 기기에 암호화 저장된 세션(`getSession`)을 읽도록 수정했습니다.

**5. 확대할수록 지도 라벨이 작아지는 역전**
SVG 전체가 스케일되니 라벨에 역스케일을 걸었더니, 이번엔 확대할수록 글씨가 작아졌습니다. 완전 고정도 완전 비례도 답이 아니어서 **지수를 중간값으로 둔 `fontSize = FONT / scale^0.6`** 으로 해결하고, 라벨 충돌 컬링도 같은 지수로 통일했습니다.
